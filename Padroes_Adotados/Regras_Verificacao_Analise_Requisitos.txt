Regras de Verificação e Análise de Requisitos

Nomenclatura dos Requisitos
-RF (Requisitos Funcionais): Descrevem as funcionalidades que o software deve fornecer e seu comportamento durante a interação com o ambiente externo.
-RNF (Requisitos Não Funcionais): Especificam as restrições sobre os serviços ou funções oferecidos pelo sistema, como desempenho, segurança, usabilidade, etc.
-Ambas siglas são acompanhadas de números que identificam unicamente os requisitos. Exemplo: [RF001] <Nome do requisito>

Regras do documento de requisitos

1- Deve se definir apenas um requisito por vez.

2- Evitar palavras: E, OU, SOMENTE SE, EXCETO, SE NECESSÁRIO, MAS, CONTUDO, ENTRETANTO, USUALMENTE, GERALMENTE, FREQUENTEMENTE, TIPICAMENTE, AMIGÁVEL,
VERSÁTIL, FLEXÍVEL, APROXIMADAMENTE, TÃO LOGO QUANTO POSSÍVEL, TALVEZ, PROVALVELMENTE ou palavras que forneçam a mesma ideia.

3-Evitar frases muito grandes. 

4- Manter a estrutura hierárquica do requisito, ou seja, encaixar um novo requisito como detalhamento de um requisito anterior. 

5- Transformar o requisitos de usuário em requisitos de software 

6- Definir requisitos funcionais para cada requisito macro ou caso de uso.
